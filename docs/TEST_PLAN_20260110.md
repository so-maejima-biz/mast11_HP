## 11. コード品質レビュー

### 11.1 シンプルさ（Simplicity）
- [ ] 不要なコードが削除されているか
- [ ] 過剰な抽象化がなく、直感的に理解できるか
- [ ] 条件分岐が複雑になりすぎていないか
- [ ] ネストが深くなりすぎていないか（3階層以下）

### 11.2 可読性（Readability）
- [ ] 変数名・関数名が意味を持っているか
- [ ] コンポーネント名がその役割を反映しているか
- [ ] CSSクラス名が命名規則（BEM風）に従っているか
- [ ] 適切なインデントとフォーマットになっているか
- [ ] マジックナンバーがなく、CSS変数を使用しているか

### 11.3 再利用性（Reusability）
- [ ] コンポーネントが適切に分離されているか
- [ ] 共通スタイルがtheme.cssに定義されているか
- [ ] 重複コードがないか
- [ ] propsやデータ構造が拡張しやすいか

### 11.4 保守性（Maintainability）
- [ ] 各コンポーネントが単一責任の原則に従っているか
- [ ] 変更の影響範囲が限定されているか
- [ ] データとビューが適切に分離されているか

### 11.5 Astroベストプラクティス
- [ ] フロントマター（---）でデータ定義がされているか
- [ ] スコープドCSSが適切に使用されているか
- [ ] data-animate属性でスクロールアニメーションが統一されているか
